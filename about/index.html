关于 python 大战 Leetcode

为什么写这个博客

首先我肯定希望通过这个博客提升自己在算法方面的能力，文字表达能力以及理性思考的能力。
Leetcode 是一个刷算法题的网站，Python 大战 Leetcode这个博客的主题就是用 Python 来挑战算法题，题目可能不仅限 Leetcode 上的算法题。
不过如果仅仅是这样的话那这个博客可以叫做 Yet another Leetcode Solutions Blog。因为网上的 Leetcode 题解实在太多了。
我的另一个目的是希望通过这个博客来帮助更多想学习编程的人更好地入门，更容易地感受到编程的美。
似乎可以总结成三个词：more, better, easier 😳。
首先解释一下 more，随着机器学习的大热，Python 编程语言似乎变成了一头风口上的猪🐷。越来越多的人开始想学习 Python，Python 语言甚至也即将进入高中教学。周围的很多朋友也时常问我有没有空教教他们编程。因此我想通过这个博客让更多的人能从零基础开始学习编程。选择 Python 语言的原因一方面是因为它的大热以及机器学习上的应用前景，另一方面是因为它真的比较适合初学者来学习。(由于 Python2 马上要退役，博客里用的都是 Python3)
这就是所谓的 better：Python 能让大多数人更好地入门。感觉大家之前都觉得编程很难很大程度上是因为以前主流的编程语言一开始的学习曲线确实十分陡峭，大学我刚开始学编程的时候就碰到这个问题，C++ 语法都还没理解的时候老师又开始教面向对象，简直是 double kill。但是 Python 不一样，脚本语言的特性让它特别方便就能运行简单的逻辑，而且很多语法跟普通英语很像，十分适合初学者理解。就像网上说的那样，“人生苦短，我用 Python” (Life is short, use Python)。让我们看一个斯坦福 CS41 -- The Python Programming Language 里面提到的一个 Python 和其他两个主流语言比较的例子。

再说一下 easier。吴恩达的机器学习课程大受欢迎就是因为他能把机器学习这个看似高深的话题讲的特别清楚。我觉得他的技巧是两个，第一个是把所有人都当成没有那么多专业知识的普通人，第二是教大家使用直觉式的理解方式，而不是艰深的数学推导去理解概念。因此我也希望自己跟随这两个原则来写这个博客。说个题外话，现在我们碰到问题都喜欢用 Google 去搜索解决方案，并不是说 Google 搜出来的比较准确，而且中文社区上好的文章太少了。看很多中文的技术博客，大多数只是为了记录某个知识点而记录，要不就是写的异常简单，简单到只有他自己或者已经懂这个点的人才能看懂，要不就是隐含了大量的前置知识没有讲清楚，默认每个人都能像博客作者一样拥有足够的前置知识点（只是我的个人观点）。反观美国的 Medium 或其他的一些技术文章，很多作者强调的都是通过类比，以直觉式地方式让一个完全不懂的新手能比较快地学会一个技术。比如有的文章标题是这样的：以横穿国家的公路旅行来解释 CSS Flexbox。当然我们现在简书上和微信公众号的很多文章也写的很好，我希望可以看到更多好文章出现在中文社区上。
我自己学东西的时候也深有感受，有时候一个技术怎么也学不懂，机缘巧合之下看了另一篇讲解的特别好的文章就突然懂了。可见教学方式或者引导学习方式的重要性。
这就是我希望自己能在这个博客里做到的。我希望提供一些直觉式的教程来让大家理解 Python 这个语言的特性和语法。然后在介绍每个技术点的时候再结合同类型的题目让大家进行练习。
自己接触编程这么多年感觉编程带给我的并不是一份有趣的工作而已，更重要的是一个理性的思维过程。我觉得每个人都应该也都有能力学会一门编程语言。

如何更好地阅读这个博客

  1. 如果想从零开始学习 Python 编程语言，建议在首页分类连接里面点击教程分类，从第一篇教程开始看起。（刚开始写博客，目前还没有教程，一有空就会加上）

  2. 如果你就想随便找些算法题做，那就在首页随便看吧，我会把每道题的链接，描述以及示例放在首页上面，看过之后可以先自己思考解法，最好能自己先写。如果做出来的话可以再点 Read More 看看我们的思路是否不同，欢迎留言把你的思路写在那道题下面。（我现在使用的是 Disqus 留言系统，有些同学不翻墙可能看不到留言系统，后续看看能不能改成其他的不需要翻墙的）。

  3. 如果你想挑选某个难度或者某种类型的题目，可以点击首页最上面的标签链接。
    我会将每篇日志根据它的 Leetcode 难度评级以及类型添加对应的标签，比如某篇日志可能会带有以下标签： Easy 和 String, 代表这题是 Easy 难度，类型跟 String 有关。
    在标签页挑选你感兴趣的类型或者难度就可以了。
